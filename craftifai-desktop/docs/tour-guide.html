1 What you get

CraftifAI ESP32 Agent bundles a chat interface, a local API (refactapi), and an AI agent (refact-lsp) with tools for ESP32 projects: build, flash, serial monitor, and related tasks — described in natural language instead of juggling many terminals.

Sign in with your CraftifAI account to start the desktop workflow.

2 First launch and setup

On first run, complete the setup flow: enter your OpenAI API key, optionally point to ESP-IDF (export.sh / Windows export script), choose a board, and pick a workspace folder for firmware projects. The app creates a local Python environment the first time; that can take a minute or two.

Fill the IDF export script, board, project directory, and sandbox path, then continue.

ESP-IDF can be configured later if you skip it at first; you need it for build and flash.

3 Chat and the agent

Use the chat to ask for changes, explain errors, or run tool actions the agent proposes. Review any confirmations before applying changes to your project. Tool availability depends on caps and your environment.

Start by typing a request in the prompt bar. Quick chips help with common tasks.

4 Workspace and sidebar

For real hardware workflows, set the IDF export script path and workspace in preferences so build and flash tools resolve the right SDK and project root. Your workspace is where new or cloned ESP-IDF projects should live.

Use the left sidebar to manage projects, chats, and switch between workspaces/groups.

5 Build and monitor flow

After generating firmware, review the assistant output and use follow-up actions like Flash to device and Monitor output. If a patch needs approval, confirm it from the prompt at the bottom.

The agent shows clear next actions when a build succeeds.

Review tool changes and grant permission when prompted.

6 Where to get help

See craftifai-desktop/README.md in the repository for system requirements, troubleshooting, and paths to logs and config (for example user settings under your OS app config directory).
